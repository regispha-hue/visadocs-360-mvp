# 📖 README — Como Usar o LDMux-OS Kernel

## 🚀 INÍCIO RÁPIDO

### 1. COPIE O ARQUIVO
- Abra: `LDMux_Kernel_v1.0_PORTABLE.md`
- Copie TODO o conteúdo (Ctrl+A, Ctrl+C)

### 2. COLE EM QUALQUER IA
- Abra nova conversa em:
  - ChatGPT (GPT-4 ou superior)
  - Claude (qualquer versão)
  - Gemini (Pro ou superior)
  - Outra IA de texto
  
- Cole o kernel completo
- Envie

### 3. AGUARDE CONFIRMAÇÃO
A IA deve responder algo como:
> "KERNEL CARREGADO. AGUARDANDO COMANDOS."

Ou similar indicando que leu e internalizou o sistema.

### 4. DÊ COMANDOS
Agora você pode dar comandos como:

**Exemplo 1 — Escrever cena:**
```
ESCREVER Monte_II "Cena do Espelho" 5-páginas POV_Quinto FOCO_Menino_Rabbit
```

**Exemplo 2 — Revisar texto:**
```
REVISAR [colar texto aqui] ASPECTO_tom_narrativo
```

**Exemplo 3 — Testar coerência:**
```
TESTAR [colar texto aqui] CONTRA_kernel
```

**Exemplo 4 — Expandir conceito:**
```
EXPANDIR Conceito_Desconciliação 3-páginas
```

## 🎯 COMANDOS DISPONÍVEIS

### ESCREVER
**Função:** Gerar narrativa nova  
**Sintaxe:** `ESCREVER [monte] [cena] [extensão] [POV] [foco]`  
**Uso:** Criar cenas, capítulos, seções

### REVISAR
**Função:** Ajustar texto existente  
**Sintaxe:** `REVISAR [texto] [aspecto]`  
**Uso:** Melhorar tom, ritmo, linguagem

### TESTAR
**Função:** Verificar coerência com kernel  
**Sintaxe:** `TESTAR [texto] CONTRA_kernel`  
**Uso:** Garantir que texto não viola leis

### EXPANDIR
**Função:** Desenvolver conceito/cena  
**Sintaxe:** `EXPANDIR [conceito] [extensão]`  
**Uso:** Aprofundar filosofia, personagens

### INTEGRAR
**Função:** Registrar aprovação  
**Sintaxe:** `INTEGRAR [texto aprovado] RUNTIME`  
**Uso:** Marcar texto como finalizado

- ✅ Dê comandos claros
- ✅ Revise outputs rigorosamente

## 📊 TESTANDO PORTABILIDADE

### TESTE BÁSICO:
Após carregar kernel, peça:

```
ESCREVER Monte_I "Semi Alado no meio-fio" 1-parágrafo POV_Quinto
```

**Output esperado:**
- Prosa poética (não dissertativa)
- Tom similar a Monte I
- Semi Alado caracterizado corretamente (uma asa, bolsa vazia)
- Sem violações de leis

**Se output for ruim:**
- Tente outra IA
- Verifique se kernel foi colado completo
- Recarregue e tente novamente

## 🔧 TROUBLESHOOTING

### PROBLEMA: IA não entendeu kernel
**Solução:** 
- Recarregue kernel completo
- Pergunte: "Você leu o kernel completo do LDMux-OS?"
- Se não, cole novamente

### PROBLEMA: Output viola leis
**Solução:**
- Use comando TESTAR
- Aponte violação específica
- Peça correção: "Isso viola Lei X, corrija"

### PROBLEMA: Tom está genérico (parece IA)
**Solução:**
- REVISAR output ASPECTO_tom_narrativo
- Referencie exemplos de Monte I no kernel
- Seja específico: "Muito dissertativo, mais imagético"

### PROBLEMA: IA quer "melhorar" kernel
**Solução:**
- Lembre: "Kernel é IMUTÁVEL"
- Não aceite "sugestões" de mudanças estruturais
- Você decide tudo, IA executa

## 🎓 EXEMPLOS DE USO

### EXEMPLO 1: Escrever cena nova

**Comando:**
```
ESCREVER Monte_II "Primeira visita do Menino Rabbit" 3-páginas POV_Quinto FOCO_Espelho_do_Si
```

**IA deve:**
1. Consultar kernel (leis, personagens, voz)
2. Gerar 3 páginas de prosa
3. Manter tom Monte I
4. Respeitar Lei do Espelho do Si
5. Caracterizar Menino Rabbit corretamente

**Você deve:**
1. Ler output
2. Verificar qualidade
3. Aprovar / Revisar / Rejeitar

---

### EXEMPLO 2: Revisar fragmento

**Comando:**
```
REVISAR [texto abaixo] ASPECTO_eliminar_tom_dissertativo

[Texto a revisar]
```

**IA deve:**
- Transformar dissertativo em imagético
- Manter conteúdo, mudar forma
- Não reescrever tudo, apenas ajustar tom

---

### EXEMPLO 3: Testar coerência

**Comando:**
```
TESTAR [texto abaixo] CONTRA_kernel

[Texto a testar]
```

**IA deve:**
- Verificar violações de leis
- Apontar inconsistências de personagens
- Sugerir correções
- NÃO aprovar/reprovar (você decide)

## 📈 WORKFLOW RECOMENDADO

### FASE 1: PREPARAÇÃO
1. Copiar kernel completo
2. Abrir conversa nova em IA escolhida
3. Colar kernel
4. Confirmar carregamento

### FASE 2: PRODUÇÃO
5. Dar comando ESCREVER
6. Receber output
7. Revisar qualidade
8. Se bom: próximo passo
9. Se ruim: REVISAR ou REJEITAR e refazer

### FASE 3: REFINAMENTO
10. Usar TESTAR para verificar coerência
11. Usar REVISAR para ajustar tom
12. Iterar até qualidade Monte I

### FASE 4: APROVAÇÃO
13. Quando satisfeito, aprovar
14. Usar INTEGRAR para registrar
15. Passar para próxima cena

## 🌟 DICAS DE OURO

### PARA MELHOR QUALIDADE:

1. **Seja específico nos comandos**
   - Ruim: "Escreva algo sobre Monte II"
   - Bom: "ESCREVER Monte_II 'Cena do Espelho' 5-páginas POV_Quinto FOCO_revelação_emocional"

2. **Referencie Monte I sempre**
   - "Mantenha padrão de qualidade Monte I"
   - "Use tom similar ao exemplo do Semi Alado no meio-fio"

3. **Itere sem medo**
   - Primeira versão raramente é perfeita
   - REVISAR quantas vezes necessário
   - Qualidade > velocidade

4. **Compare IAs**
   - Mesma cena em Claude vs GPT vs Gemini
   - Veja qual mantém voz melhor
   - Use melhor IA para cada tipo de cena

5. **Documente decisões**
   - Quando aprovar algo, registre por quê
   - Quando rejeitar, registre o que estava errado
   - Aprenda com processo

## 📞 SUPORTE

### SE TIVER PROBLEMAS:

**Kernel não funciona:**
- Verifique se colou completo
- Tente IA diferente
- Recarregue em conversa nova

**Output sempre ruim:**
- Talvez IA não seja capaz (use GPT-4+ ou Claude)
- Talvez comandos não estejam claros
- Talvez kernel precise refinamento (v1.1)

**Dúvidas sobre leis:**
- Releia seção de Leis Fundamentais
- Use comando TESTAR para verificar
- Quando em dúvida, peça à IA explicar lei
