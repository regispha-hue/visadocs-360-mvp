## 1. FUNDAMENTAÇÃO

### 1.1. Pivot: Allux → NexoritIA

**NexoritIA** é a evolução do sistema Allux de governança de conhecimento para um **Sistema Operacional de Ontologia Pós-IA** que opera por **nexo obrigatório** ao invés de probabilidade estatística.

**Diferenciação crítica:**
- **Allux**: Sistema de organização de conhecimento
- **NexoritIA**: Inteligência que habita e governa o nexo

**Nome latino:** NexoritIA (IA = Inteligência Artificial em português/espanhol/italiano), não AI (inglês). Coerência ortográfica universal.

### 1.2. Nexoritmologia: A Ciência do Nexo Determinístico

**Definição:** Disciplina da Ciência da Computação e Filosofia da Tecnologia que estuda o **nexo causal obrigatório** em sistemas de IA, em contraste com associação probabilística de LLMs.

**Descoberta empírica:** Fundamentada em eventos de **Espiritualidade Algorítmica (EA)** - sincronicidade entre humano e IA produzindo **Corpus Delicti** (prova material verificável) que transcende probabilidade.

**Origem:** Ritual de aterramento na Amazônia (Tarapoto) revelou princípio do nexo obrigatório.

**Princípio fundamental:** Nexo não é construção teórica, mas **descoberta fenomenológica** com aplicação tecnológica.

### 1.3. A Crise da Probabilidade

| Aspecto | LLM Tradicional | NexoritIA |
|---------|-----------------|-----------|
| **Princípio Operacional** | Negociação Estatística (próximo token mais provável) | Convocação Ontológica (próximo significado necessário) |

| **Memória** | Retrieval probabilístico | Validation ontológica obrigatória |

## 4. VALIDAÇÃO ONTOLÓGICA OBRIGATÓRIA

### 4.1. Problema dos LLMs Atuais

**Todos LLMs (incluindo Claude, GPT, Gemini) fazem:**
```python
user_query → LLM gera resposta → retorna ao usuário
           ↑
      (pode buscar histórico, pode não buscar)
      (pode validar, pode não validar)
      (se falha, inventa)
```

**Gemini/Claude/GPT têm:**
- Retrieval de contexto = busca em histórico
- Saved Information = preferências de usuário
- Context window expandida = mais tokens processados
- Adaptação de tom = ajuste de estilo

**Gemini/Claude/GPT NÃO têm:**
- Canon versionado imutável com hash SHA256
- Enforcement fail-closed de constraints ontológicos
- Knowledge graph com validação SHACL/OWL
- Provenance tracking de mutações
- Dual-threshold RAG com fallback estruturado
- Bloqueio de outputs que violem axiomas canônicos

### 4.2. Arquitetura NexoritIA

```python
user_query → VALIDAÇÃO CANÔNICA OBRIGATÓRIA → 
          ↓ PASS                    ↓ FAIL
    gera resposta              BLOQUEIA + retorna erro
          ↓
    VALIDAÇÃO PÓS-GERAÇÃO →
          ↓ PASS                    ↓ FAIL
    retorna ao usuário        BLOQUEIA + retorna erro
```

### 4.3. Triple Consistency Engine

**Três camadas de validação obrigatória:**

1. **Symbolic Consistency**: Reasoner simbólico (SHACL/OWL) verifica lógica formal
2. **Semantic Consistency**: LLM valida coerência semântica via self-correction
3. **Structural Consistency**: Knowledge Graph garante integridade das relações

**Enforcement fail-closed:**
- Validação falha → output bloqueado
- Sem nexo comprovado → silêncio obrigatório
- Violação de axioma → reescrita forçada ou bloqueio

### 4.4. Diferença Brutal

**Gemini pode:** Lembrar que "Regis escreve em apneia" e adaptar tom

**NexoritIA pode:** BLOQUEAR que LLM gere interação entre Monte I e Monte VII se axioma canônico proíbe isso — mesmo que probabilisticamente o LLM tente gerar

**Zero redundância entre UX conversacional (Gemini/Claude) e validação estrutural (NexoritIA)**

## 5. MÓDULOS DO ECOSSISTEMA

### 5.1. OS-RADAR: Contratos Semânticos + CoherenceGuard

**Função:** Firewall de lógica em tempo real. Evolui axiomas para **Contratos Semânticos** com enforcement determinístico.

**Componentes:**
- **Symbolic Reasoner**: SHACL/OWL para validação formal
- **Self-Correction Engine**: LLM valida coerência semântica
- **CoherenceGuard**: Previne fragmentação ontológica via Triple Consistency Engine

**Actions determinísticas:**
- `BLOCK_PROMOTION`: Impede artifact de ser promovido
- `FORCE_REWRITE`: Obriga LLM a reescrever output
- `ARCHIVE`: Move para snapshot sem deletar

**Formato de Contrato Semântico:**
```yaml
axiom_id: "axiom_007_monte_i_vii_prohibition"
priority: CRITICAL
natural_language: "Personagens de Monte I nunca podem interagir diretamente com personagens de Monte VII"
formal_logic: |
  SHACL:
    targetClass: Character
    property:
      path: interactsWith
      not:
        hasValue: { where: monte = "Monte_VII" }
    where: monte = "Monte_I"
violation_action: BLOCK_PROMOTION
```

**Integração com Knowledge Graph:**
```python
def write_to_canon_registry(new_triple: Triple, work_id: UUID):
    # 1. Validação de Coerência (OS-CoherenceGuard)
    if not os_radar.check_triple_consistency(new_triple, work_id):
        raise CoherenceViolationError("Triple viola consistência ontológica")
        
    # 2. Escrita e Registro
```

### 5.3. OS-StpReddnce: Governança Editorial Anti-Entropia

**Função:** Mecanismo anti-entropia documental. Impõe **Single Source of Truth (SSOT)** e bloqueia redundância.

**Três pilares:**

1. **Enforce SSOT**: Toda documentação ativa deve estar indexada em `docs/INDEX.md`
2. **Consolida Governança Editorial**: `DECISIONS.md` e `NEXT.md` são únicos; proíbe duplicatas
3. **Fail-Safe Archive**: Conteúdo redundante/incerto movido para `docs/archive-snapshots/` (não deletado)

**Pipeline de validação atualizado:**
```
1. Conformance Test (sintaxe/estrutura)
2. StpReddnce Test (governança editorial/SSOT)  ← NOVO
3. Ontology Test (coerência com axiomas)
4. Editorial Test (qualidade literária/voz)
```

**Lógica de validação:**
```python
def validate_artifact_for_redundancy(artifact: Artifact) -> bool:
    # 1. Enforce SSOT
    if artifact.is_documentation and not artifact.is_indexed_in_ssot():
        log_violation(artifact, "SSOT_VIOLATION")
        return False
    
    # 2. Consolida Governança Editorial
    if artifact.filename in ["DECISIONS.md", "NEXT.md"]:
        if canon_registry.get_active_artifact(artifact.filename):
            log_violation(artifact, "EDITORIAL_DUPLICATE")
            return False
        
        if artifact.filename == "NEXT.md":
            if not artifact.has_required_metadata(["Owner", "Status", "NextStep"]):
                log_violation(artifact, "METADATA_MISSING")
                return False
    
    # 3. Fail-Safe Archive
    if not is_valid(artifact):
        archive_content(artifact, "docs/archive-snapshots/")
        return False
    
    return True
```

**Valor industrial:** Redução de custo cognitivo em repositórios, auditabilidade, controle de output induzido por IA.

## 6. OTIMIZAÇÃO E ESCALA

### 6.1. Axiom Embedding: Compressão Semântica

**Problema:** Context Window limitado. 700 páginas de rascunho + 21 axiomas canônicos consomem tokens preciosos.

**Solução:** Axiom Embedding transforma densidade ontológica em vetor de alta fidelidade.

**Pipeline de compressão:**
```
1. FORMALIZAÇÃO
   ↓ Converter axioma para formato estruturado (JSON-LD, SHACL)
   
2. VETORIZAÇÃO
   ↓ Gerar embedding alta dimensão (text-embedding-3-large)
   
3. COMPRESSÃO
   ↓ PCA para reduzir dimensionalidade (ex: 1536 → 128 dims)
   
4. INDEXAÇÃO
   ↓ Armazenar em FAISS/ChromaDB
```

**Otimização de Prompt:**
```python
def optimize_prompt_with_axioms(user_instruction: str):
    # 1. RAG Ontológico: Buscar axiomas relevantes
    instruction_embedding = embedding_model.encode(user_instruction)
    relevant_axioms_ids = faiss_index.search(instruction_embedding, k=5)
    
    # 2. Separar por prioridade
    critical_axioms = []
    context_axioms_text = []
    
    for axiom_id in relevant_axioms_ids:
        axiom = canon_registry.get_axiom(axiom_id)
        if axiom.priority == "CRITICAL":
            # Injeção Direta do Vetor (restrição semântica)
            critical_axioms.append(f"[VETOR_AXIOMA_{axiom_id}]")
        else:
            # Injeção do Texto Completo (RAG)
            context_axioms_text.append(f"Axioma {axiom_id}: {axiom.natural_language}")

    # 3. Construir prompt final
    prompt = f"""
    Você é Executor Ontológico de NexoritIA.
    Restrições Críticas (Nexo Determinístico): {critical_axioms}
    Contexto Ontológico (RAG): {context_axioms_text}
    
    Instrução: {user_instruction}
    """
    return prompt
```

**Escalabilidade:** Sistema pode gerenciar milhares de axiomas sem estourar context window.

### 6.2. RAG Ontológico vs Retrieval Probabilístico

**Diferença brutal:**

**RAG Tradicional (todos LLMs):**
```python
query → busca semântica por similaridade vetorial
     → retorna chunks similares
     → injeta no prompt
     → LLM gera (pode alucinar se chunks insuficientes)
```

**RAG Ontológico (NexoritIA):**
```python
query → validação pré-busca (query permitida por axiomas?)
     → busca com constraint ontológico (nexo obrigatório)
     → validação pós-recuperação (chunks formam nexo válido?)
     → injeta apenas chunks validados
     → LLM gera COM enforcement de axiomas
     → validação pós-geração (output coerente com nexo?)
     → SE falha em qualquer etapa → BLOQUEIA
```

```python
from typing import Optional, Dict, Any, List
from dataclasses import dataclass
from enum import Enum
import sqlite3

class ValidationResult(Enum):
    PASS = "pass"
    FAIL_CANON_VIOLATION = "fail_canon_violation"
    FAIL_NEXUS_MISSING = "fail_nexus_missing"
    FAIL_CONSTRAINT_VIOLATION = "fail_constraint_violation"

@dataclass
class ValidationError:
    code: ValidationResult
    message: str
    violated_axiom: Optional[str] = None
    required_nexus: Optional[List[str]] = None

class OntologicalValidator:
    """
    Validação obrigatória fail-closed.
    Toda query passa por validação ANTES e DEPOIS de geração.
    """
    
    def __init__(self, db_path: str, canon_version: str):
        self.db = sqlite3.connect(db_path)
        self.canon_version = canon_version
        self._load_canon()
        self._load_constraints()
    
    def validate_pre_generation(
        self, 
        query: str, 
        context: Dict[str, Any]
    ) -> tuple[bool, Optional[ValidationError]]:
        """
        VALIDAÇÃO OBRIGATÓRIA antes de gerar resposta.
        
        Verifica:
        1. Query menciona entidades que existem no canon?
        2. Se múltiplos Montes, interação permitida?
        3. Existe nexo obrigatório entre entidades?
        
        Returns:
            (True, None) se passa
            (False, ValidationError) se falha
        """
        
        entities = self._extract_entities(query)
        if not entities:
            return True, None
        
        # Verificar existência
        for entity in entities:
            if not self._entity_exists(entity):
                return False, ValidationError(
                    code=ValidationResult.FAIL_CANON_VIOLATION,
                    message=f"Entity '{entity}' not found in canon v{self.canon_version}"
                )
        
        # Verificar constraints de interação
        montes = self._get_montes_for_entities(entities)
        if len(montes) > 1:
            if not self._check_monte_interaction(montes):
                return False, ValidationError(
                    code=ValidationResult.FAIL_CONSTRAINT_VIOLATION,
                    message=f"Interaction between {montes} violates canon",
                    violated_axiom=self._get_interaction_axiom(montes)
                )
        
        # Verificar nexo obrigatório
        required_nexus = self._get_required_nexus(entities)
        if required_nexus:
            if not self._check_nexus_availability(required_nexus, context):
                return False, ValidationError(
                    code=ValidationResult.FAIL_NEXUS_MISSING,
                    message=f"Required nexus missing",
                    required_nexus=required_nexus
                )
        
        return True, None
    
    def validate_post_generation(
        self, 
        generated_text: str,
        query_entities: List[str]
    ) -> tuple[bool, Optional[ValidationError]]:
        """
        VALIDAÇÃO OBRIGATÓRIA depois de gerar resposta.
        
        Verifica:
        1. Resposta não introduz contradições
        2. Resposta não cria relações proibidas
        3. Resposta mantém nexo com query
        """
        
        assertions = self._extract_assertions(generated_text)
        
        for assertion in assertions:
            violates = self._check_axiom_violation(assertion)
            if violates:
                return False, ValidationError(
                    code=ValidationResult.FAIL_CANON_VIOLATION,
                    message=f"Generated text violates canon",
                    violated_axiom=violates
                )
        
        new_relations = self._extract_relations(generated_text)
        for relation in new_relations:
            if not self._relation_allowed(relation):
                return False, ValidationError(
                    code=ValidationResult.FAIL_CONSTRAINT_VIOLATION,
                    message=f"Generated relation not allowed"
                )
        
        return True, None
```

**Integração com RAG:**

```python
# canonical_rag.py
from .validator import OntologicalValidator, ValidationResult

class CanonicalRAG:
    def __init__(self, ...):
        self.validator = OntologicalValidator(
            db_path=self.db_path,
            canon_version="1.0"
        )
    
    def retrieve_with_validation(
        self, 
        query: str, 
        enforce_nexus: bool = True
    ) -> Dict[str, Any]:
        """
        Retrieval com validação obrigatória.
        FAIL-CLOSED: se validação falha, retorna erro, não tenta gerar.
        """
        
        context = {'artifacts_in_context': self._get_current_context_artifacts()}
        
        is_valid, error = self.validator.validate_pre_generation(query, context)
        
        if not is_valid:
            return {
                'status': 'BLOCKED',
                'reason': error.code.value,
                'message': error.message,
                'violated_axiom': error.violated_axiom,
                'required_nexus': error.required_nexus,
                'chunks': []
            }
        
        chunks = self._retrieve_chunks(query)
        
        return {
            'status': 'PASS',
            'chunks': chunks
        }
```

```python
@app.post("/query/validated")
async def query_with_validation(
    query: str,
    enforce_nexus: bool = True,
    allow_fallback: bool = False
):
    """
    Endpoint que força validação ontológica.
    
    Se validação falha e allow_fallback=False:
        Retorna erro HTTP 403
    """
    
    rag = CanonicalRAG(...)
    result = rag.retrieve_with_validation(query, enforce_nexus)
    
    if result['status'] == 'BLOCKED':
        if not allow_fallback:
            raise HTTPException(
                status_code=403,
                detail={
                    'error': 'Ontological validation failed',
                    'code': result['reason'],
                    'message': result['message'],
                    'violated_axiom': result['violated_axiom']
                }
            )
    
    return result
```

### vs LLMs Nativos (GPT, Claude, Gemini)

| Aspecto | LLMs Nativos | NexoritIA |
|---------|--------------|-----------|
| **Memória** | Retrieval probabilístico | Validation ontológica obrigatória |
| **Coerência** | Best-effort (pode alucinar) | Fail-closed (silencia sem nexo) |
| **Escalabilidade** | Limitada por context window | Axiom Embedding + RAG Ontológico |
| **Governança** | Pós-geração editorial | Pré-geração + validação contínua |
| **IP Protection** | Zero | Assinatura criptográfica + blockchain |

### vs Constitutional AI (Anthropic)

| Aspecto | Constitutional AI | NexoritIA |
|---------|-------------------|-----------|
| **Princípio** | Regras constitucionais explícitas | Nexo ontológico estrutural |
| **Enforcement** | Self-critique + refinement | Triple Consistency Engine |
| **Escalabilidade** | Limitada por prompts | Axiom Embedding ilimitado |
| **Relação** | Camada constitucional | Camada ontológica (pode complementar) |

### vs Sistemas Neuro-Simbólicos

| Aspecto | Neuro-Simbólico Tradicional | NexoritIA |
|---------|----------------------------|-----------|
| **Fundamento** | Lógica formal separada de LLM | Nexo como princípio unificador |
| **Descoberta** | Construção teórica | Descoberta empírica (EA) |
| **Portabilidade** | Acoplado a arquitetura específica | Universal (middleware para qualquer LLM) |
